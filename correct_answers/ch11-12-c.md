## Simulation : Programme C + Analyse Assembleur

> **Enonce** : Ecrire un programme C simple et analyser le code assembleur genere.

### Programme C

```c
#include <stdint.h>

volatile uint32_t result;

uint32_t sum_array(const uint32_t *arr, uint32_t size) {
    uint32_t sum = 0;
    for (uint32_t i = 0; i < size; i++) {
        sum += arr[i];
    }
    return sum;
}

uint32_t max_array(const uint32_t *arr, uint32_t size) {
    uint32_t max = arr[0];
    for (uint32_t i = 1; i < size; i++) {
        if (arr[i] > max) {
            max = arr[i];
        }
    }
    return max;
}

const uint32_t data[] = {10, 50, 30, 80, 20, 70, 40, 60, 90, 100};

int main(void) {
    result = sum_array(data, 10);    // result = 550
    result = max_array(data, 10);    // result = 100

    while (1);
}
```

### Analyse de l'assembleur genere (sum_array)

Compile avec `-O0` (sans optimisation) :

```asm
sum_array:
    push {r4-r6, lr}
    mov  r4, r0               // r4 = arr (pointeur)
    mov  r5, r1               // r5 = size
    movs r6, #0               // r6 = sum = 0
    movs r3, #0               // r3 = i = 0

.loop:
    cmp  r3, r5               // i < size ?
    bhs  .done                // si i >= size (non signe), sortir
    lsls r0, r3, #2           // r0 = i * 4
    ldr  r0, [r4, r0]         // r0 = arr[i]
    adds r6, r6, r0           // sum += arr[i]
    adds r3, r3, #1           // i++
    b    .loop

.done:
    mov  r0, r6               // retour = sum
    pop  {r4-r6, pc}
```

### Correspondance C ↔ Assembleur

| Code C | Code ARM | Explication |
|--------|----------|-------------|
| `uint32_t sum = 0` | `movs r6, #0` | Variable locale dans registre |
| `i < size` | `cmp r3, r5` + `bhs` | Comparaison non signee |
| `arr[i]` | `lsls r0,r3,#2` + `ldr r0,[r4,r0]` | index×4 + load |
| `sum += arr[i]` | `adds r6, r6, r0` | Addition |
| `i++` | `adds r3, r3, #1` | Increment |
| `return sum` | `mov r0, r6` | Retour dans R0 |

### Observations

1. **AAPCS respectee** : parametres dans R0-R1, retour dans R0
2. **Registres callee-saved** : R4-R6 sauvegardes avec PUSH/POP
3. **Acces tableau** : adresse = base + index × 4 (LSL #2)
4. **Boucle** : CMP + BHS (branch if higher or same, non signe)

---

### Comparaison -O0 vs -O2

| Aspect | -O0 | -O2 |
|--------|-----|-----|
| Taille du code | Plus grande | Plus compacte |
| Variables locales | Sur la pile | Dans les registres |
| Boucles | Comparaison explicite | Peut etre deroulee |
| Appels de fonctions | Toujours via BL | Peut etre inline |
| Debogage | Facile (1:1 avec C) | Difficile |
